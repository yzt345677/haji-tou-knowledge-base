🛠️ AI Agent 工具使用与函数调用

从工具定义到函数调用的完整流程。掌握工具分类、参数设计、错误处理、权限控制，
实现 AI Agent 与外部系统的安全可靠集成。

📦 工具分类

📁
文件操作类
FileRead, FileWrite, FileEdit, Glob, Grep
用途：读取、写入、编辑文件，文件搜索
💻
Shell 执行类
Bash, PowerShell, REPL
用途：执行系统命令、脚本、交互式会话
🌐
网络请求类
WebSearch, WebFetch, Http
用途：搜索网页、抓取内容、API 调用
🔌
MCP 集成类
MCP, ListMcpResources, ReadMcpResource
用途：Model Context Protocol 集成
📋
任务管理类
TaskCreate, TaskList, TaskOutput, TaskStop
用途：创建、管理、监控子任务
🤖
Agent 协作类
Agent, SendMessage, TeamCreate
用途：启动子 Agent、发送消息、团队协作

📐 工具定义规范

typescript
const inputSchema = z.strictObject({
command: z.string().describe('The command to execute'),
timeout: z.number().optional().describe('Optional timeout in milliseconds'),
description: z.string().optional().describe('Clear description of what this command does'),
run_in_background: z.boolean().optional().describe('Run command in background'),
})
参数 类型 必填 说明
command string ✅ 要执行的命令
timeout number ❌ 超时时间（毫秒）
description string ❌ 命令描述（用于权限审批）
run_in_background boolean ❌ 是否后台运行

🔄 函数调用流程

typescript
async call(args, context) {
// 1. 验证输入参数
const validationResult = await validateInput(args, context);
if (!validationResult.valid) {
throw new Error(validationResult.reason);
}
// 2. 权限检查
const permission = await checkPermissions(args, context);
if (permission.decision === 'deny') {
throw new Error('Permission denied');
}
// 3. 执行工具
const result = await executeTool(args);
// 4. 返回结果
return {
data: result,
newMessages: []
};
}
✅
步骤 1：验证输入
使用 Zod schema 验证参数类型和格式
确保必填字段存在、格式正确
🔐
步骤 2：权限检查
检查用户是否有权限执行此操作
可能需要用户确认（ask 模式）
⚡
步骤 3：执行工具
调用实际的工具实现
捕获异常并处理错误
📤
步骤 4：返回结果
返回结构化结果
可选添加新消息到对话

⚠️ 错误处理最佳实践

错误类型 处理方式 示例
参数验证失败 返回详细错误信息 "Missing required field: command"
权限拒绝 提示用户申请权限 "Permission denied. Please contact admin."
工具执行失败 返回错误原因 + 建议 "File not found. Did you mean: ./src/index.ts?"
超时 提示用户重试或增加超时 "Command timed out after 300s. Try increasing timeout."
资源不存在 提供替代方案 "MCP server 'github' not found. Available: gitlab"
