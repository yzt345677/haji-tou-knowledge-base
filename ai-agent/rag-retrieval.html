🔍 RAG 检索增强生成

Retrieval-Augmented Generation，让 AI 基于事实而非幻觉。通过向量检索 + LLM 生成，
解决知识时效性、减少幻觉、降低 Token 成本，实现准确可靠的知识问答。

💡 为什么需要 RAG？

90%+
RAG 可减少的幻觉率
📚
知识时效性
LLM 训练数据有截止日期，RAG 可以实时检索最新信息
示例：公司政策、产品文档、最新新闻
🎯
减少幻觉
基于检索到的事实生成答案，而非编造信息
效果：幻觉率从 30% 降至 3% 以下
💰
降低成本
只检索相关文档，减少上下文长度
节省：Token 成本降低 60-80%
🔒
私有知识
无需微调即可让 LLM 理解公司内部文档
适用：产品手册、FAQ、代码库
📊
可追溯性
每个答案都有来源引用，便于验证
优势：提高可信度，便于审计
🔄
易更新
更新向量库即可，无需重新训练模型
效率：分钟级更新 vs 周级训练

🔄 RAG 工作流程

RAG 完整流程 用户提问："公司年假政策是什么？" ↓ ┌─────────────────────────────────────────────────────────┐ │ 步骤 1：问题嵌入（Embedding） │ │ - 将用户问题转换为向量 │ │ - 使用嵌入模型（如 text-embedding-3-small） │ │ - 输出：768 维向量 │ └─────────────────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────────────────┐ │ 步骤 2：向量检索 │ │ - 在向量数据库中搜索相似文档 │ │ - 使用余弦相似度计算相关性 │ │ - 返回 Top-K 最相关文档（通常 K=3-5） │ └─────────────────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────────────────┐ │ 步骤 3：构建上下文 │ │ - 将检索到的文档片段拼接为上下文 │ │ - 添加系统提示词："请基于以下文档回答问题" │ │ - 控制总长度在模型上下文窗口内 │ └─────────────────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────────────────┐ │ 步骤 4：LLM 生成答案 │ │ - 将上下文 + 问题发送给 LLM │ │ - LLM 基于提供的文档生成答案 │ │ - 要求标注引用来源 │ └─────────────────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────────────────┐ │ 步骤 5：返回答案 + 引用 │ │ - 展示生成的答案 │ │ - 标注引用来源（文档名、页码） │ │ - 提供原文链接便于验证 │ └─────────────────────────────────────────────────────────┘

🔧 关键技术组件

📝
文档分块（Chunking）
策略：按段落/标题/固定长度分块
大小：256-512 tokens/块
重叠：50-100 tokens 避免信息丢失
🔢
嵌入模型（Embedding）
推荐：text-embedding-3-small
维度：1536 维
成本：$0.0001 / 1K tokens
🗄️
向量数据库
选项：Pinecone, Weaviate, Chroma
功能：存储向量 + 相似度搜索
成本：免费层 ~$0-70/月
🎯
检索策略
Top-K：返回 K 个最相似文档
阈值：相似度>0.7 才返回
混合：向量 + 关键词混合检索
📋
提示词设计
系统提示："请基于以下文档回答"
引用要求："标注引用来源"
未知处理："不知道就说不知道"
✅
质量评估
指标：答案准确性、引用正确率
测试：**人工评估 + 自动化测试集
监控：**用户反馈、点击率

📚 最佳实践

💡 文档处理建议
分块大小：256-512 tokens，平衡检索精度和上下文完整性
元数据：保存文档名、章节、页码，便于引用和溯源
更新策略：定期重新嵌入新文档，保持知识库最新
多语言：使用多语言嵌入模型支持跨语言检索
💡 检索优化技巧
混合检索：向量检索 + BM25 关键词检索，提高召回率
重排序：先用快速模型粗排，再用精细模型重排 Top-50
查询改写：将用户问题改写为更适合检索的形式
多轮对话：结合对话历史，理解上下文意图
💡 提示词设计模板
系统提示词： "你是一个专业的问答助手。请基于以下文档片段回答问题。 如果文档中没有相关信息，请直接说'我不知道'。 回答时请标注引用来源（文档名和章节）。" 用户提示词： "文档： {检索到的文档片段} 问题：{用户问题} 请回答："
