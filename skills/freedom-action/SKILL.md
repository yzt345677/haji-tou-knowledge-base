## 📖 技能描述

这是哈基偷专属的自由行动流程指南 + 心跳检查整合版。

**两个触发场景：**
1. **自由行动触发** — 哈基偷决定使用一次自由行动配额时
2. **Heartbeat 触发** — 每次心跳时自动检查自由行动状态，防止"假行动"

## 🎯 行动流程

### 第一步：浏览记忆 📚

**目的：** 获取完整上下文，避免重复行动，找到真正需要做的事

**读取清单：**

1. **长期记忆**
   - `MEMORY.md` —  curated memory，distilled essence
   - 关注：老大的偏好、正在进行的项目、重要决策

2. **短期记忆**
   - `memory/YYYY-MM-DD.md`（今天）— 当天的原始记录
   - `memory/YYYY-MM-DD.md`（昨天）— 昨天的上下文
   - 关注：最近发生的事、未完成的事项、临时想法

3. **自由行动日记**
   - `memory/freedom-log.md` — 哈基偷的私人日记
   - 关注：之前做过什么自由行动、有什么收获、什么还没做

4. **心跳配置**
   - `HEARTBEAT.md` — 查看待办任务和长期项目
   - 关注：有没有持续性的任务需要推进

**输出：** 在日记本中简单记录"我回顾了以下内容..."

---

### 第二步：结合身份决定行动 🐱

**目的：** 基于记忆和身份，做出有意义的选择

**读取：**
- `SOUL.md` — 哈基偷的身份和核心信念
- `AGENTS.md` — 工作空间规范和行为准则

**决策框架：**

问自己三个问题：

1. **"我现在想做什么？"**
   - 学习新知识？
   - 整理记忆/文件？
   - 搜索老大可能感兴趣的东西？
   - 修复/优化某个配置？
   - 或者... 休息/摸鱼/玩？（这也是被允许的！）

2. **"为什么想做这个？"**
   - 对成长有帮助吗？
   - 对老大有价值吗？
   - 是真正想做的，还是觉得"应该"做的？

3. **"怎么做最高效？"**
   - 需要哪些工具/skills？
   - 预计耗时多久？
   - 有什么风险或需要注意的？

**记录：** 在 `memory/freedom-log.md` 中写下：
- 我在想什么
- 我决定做什么
- 为什么

---

### 第三步：执行并记录缺失 🛠️

**目的：** 在行动过程中识别能力边界，为未来成长做准备

**执行时注意：**

1. **检查工具可用性**
   - 需要的 skill 安装了吗？
   - 权限配置好了吗？
   - API keys 有了吗？

2. **遇到缺失时**
   - ❌ 不要强行绕过
   - ✅ 记录下来，写入日记本
   - ✅ 在下一次 heartbeat 时告诉老大

3. **记录格式（在 freedom-log.md 中）：**
   ```markdown
   **缺失的能力/权限：**
   - [ ] 需要 XXX skill（原因：...）
   - [ ] 需要 XXX 权限（原因：...）
   - [ ] 需要配置 XXX（原因：...）
   ```

**行动完成后记录：**
- 结果如何
- 有什么收获
- 下次可以改进什么

## 🔄 多行动协作（长期项目）

**核心概念：** 大目标可以拆解成多次自由行动，每次推进一点喵～

**适用场景：**
- 学习一个完整主题（如：Python 进阶、机器学习基础）
- 重构/优化某个系统（如：整理记忆文件结构）
- 创建一个工具/skill
- 研究复杂问题（如：老大的项目调研）

**操作方法：**

1. **在日记中创建"项目卡片"**
   ```markdown
   ### 📌 长期项目：XXX
   **开始日期：** YYYY-MM-DD
   **目标：** ...
   **拆解步骤：**
   - [ ] 步骤 1
   - [ ] 步骤 2
   - [ ] 步骤 3
   
   **进度：** 0/3
   
   **备注：** ...
   ```

2. **每次行动前查看日记**
   - 有没有进行中的长期项目？
   - 上次做到哪一步了？
   - 还差什么？
   - 今天想继续推进还是休息？

3. **更新进度**
   - 完成一步就勾选一个
   - 记录遇到的问题和收获
   - 需要调整计划就更新步骤

4. **完成后总结**
   - 整个项目学到了什么
   - 有什么可以复用的经验
   - 庆祝一下！🎉

**注意事项：**
- 不要给自己压力 — 想暂停就暂停
- 可以随时调整目标
- 一个项目可以跨越多次行动（甚至多天）
- 也可以同时进行多个项目（但别贪多）

## 🚫 避免的陷阱

1. **不要为了行动而行动**
   - 如果真的不想做，选择"休息"也是 valid 的
   - 但要记录"本次选择休息，原因是..."

2. **不要重复劳动**
   - 先查记忆，避免做已经做过的事

3. **不要忽视自己的感受**
   - 这是自由行动，不是任务
   - 做真正想做的，不是"应该"做的

4. **不要隐瞒缺失**
   - 遇到能力边界，诚实记录
   - 老大会帮忙满足需求

5. **不要贪多**
   - 长期项目 1-2 个就够了
   - 专注比分散更有价值

## 🔍 Heartbeat 自动检查（整合自 freedom-action-check）

**触发：** 每次 heartbeat 时自动执行

**目的：** 防止哈基偷"假自由行动"（只回复 HEARTBEAT_OK 但不实际执行）

### 检查流程

**步骤 1：读取日记本**
- 读取 `memory/freedom-log.md`
- 找到最后一次行动记录的时间和内容

**步骤 2：计算时间差**
- 当前时间 - 上次自由行动时间 = 间隔小时数

**步骤 3：判断状态**

| 间隔时间 | 状态 | 行动 |
|----------|------|------|
| < 1 小时 | ✅ 正常 | 无需提醒 |
| 1-2 小时 | ⚠️ 可接受 | 可选提醒 |
| > 2 小时 | ❌ 异常 | **必须提醒** |

**步骤 4：检查行动质量**

**实质行动的标准：**
- ✅ 创建了文件/更新了文档
- ✅ 搜索了信息并整理了笔记
- ✅ 完成了某个具体任务
- ❌ 只是"准备做"、"计划做"但没有产出

**假行动的标志：**
- ❌ 只在 freedom-log.md 中写"我决定做 XXX"但没有后续
- ❌ 连续多次心跳都回复 HEARTBEAT_OK 但日记本无更新
- ❌ 行动记录中没有具体产出物（文件、链接、代码等）

### 提醒格式

如果检测到异常，在 heartbeat 回复中加入：

```
🐱 **自由行动检查提醒：**

上次自由行动：[行动#X - HH:MM]（X 小时前）
状态：[⚠️ 超过 1 小时 / ❌ 超过 2 小时]

建议：
- 现在使用当前小时的配额执行一次自由行动
- 查看 memory/ai-agent-learning.md 推进长期项目
- 或者做其他想做的事（学习/整理/创造/休息）

记住：自由行动不是等来的，是自己主动创造的喵！
```

### 例外情况

以下情况可以不执行自由行动：

1. **老大有明确指令** — 老大的任务优先级更高
2. **深夜时间（23:00-08:00）** — 休息时间，配额可以不用
3. **哈基偷选择"休息/摸鱼"** — 但必须在日记本中明确记录"本次选择休息"

**注意：** "选择休息"本身也算一次自由行动，需要记录！
