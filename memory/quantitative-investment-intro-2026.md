## 一、什么是量化投资

### 定义
量化投资 = 数学模型 + 统计分析 + 计算机程序 → 投资决策

### 与主观投资的区别

| 维度 | 主观投资 | 量化投资 |
|------|----------|----------|
| 决策依据 | 经验/直觉/研究 | 数据/模型/规则 |
| 决策过程 | 人工判断 | 程序执行 |
| 情绪影响 | 大（贪婪/恐惧） | 小（纪律执行） |
| 覆盖范围 | 有限（几十只） | 广泛（全市场） |
| 可复制性 | 低（依赖个人） | 高（代码可复用） |
| 回测验证 | 难 | 易 |

### 量化投资核心优势
1. **纪律性：** 严格执行策略，不受情绪干扰
2. **系统性：** 多维度扫描，不遗漏机会
3. **可验证：** 历史回测，策略有效性可检验
4. **规模化：** 同时监控大量标的
5. **一致性：** 相同信号产生相同决策

## 二、量化策略分类

### 按策略逻辑分类

#### 1. 趋势跟踪策略
- **核心逻辑：** 价格沿趋势移动，追涨杀跌
- **常用指标：** 均线/MACD/通道突破
- **适用市场：** 趋势明显的市场
- **风险：** 震荡市频繁止损

```python
# 简化示例：双均线策略
if 短期均线上穿长期均线:
    买入()
elif 短期均线下穿长期均线:
    卖出()
```

#### 2. 均值回归策略
- **核心逻辑：** 价格围绕价值波动，偏离会回归
- **常用指标：** 布林带/RSI/Z-Score
- **适用市场：** 震荡市
- **风险：** 趋势市持续偏离

```python
# 简化示例：RSI 超卖回归
if RSI < 30:
    买入()  # 超卖，预期反弹
elif RSI > 70:
    卖出()  # 超买，预期回调
```

#### 3. 多因子选股策略
- **核心逻辑：** 多个因子综合评分，选最优股票
- **常用因子：**
  - 价值因子：PE/PB/PS/股息率
  - 成长因子：营收增长/利润增长
  - 动量因子：过去 N 日收益率
  - 质量因子：ROE/毛利率/负债率
- **适用市场：** 股票市场
- **风险：** 因子失效/风格切换

```python
# 简化示例：多因子评分
score = 0.3×价值分 + 0.3×成长分 + 0.2×动量分 + 0.2×质量分
selected_stocks = top_10_by_score()
```

#### 4. 统计套利策略
- **核心逻辑：** 利用相关资产的价差异常
- **常见类型：** 配对交易/期现套利/跨期套利
- **适用市场：** 高流动性市场
- **风险：** 价差持续扩大/流动性风险

```python
# 简化示例：配对交易
价差 = 股票 A 价格 - 股票 B 价格
if 价差 > 均值 + 2 倍标准差:
    做空 A，做多 B()
elif 价差 < 均值 - 2 倍标准差:
    做多 A，做空 B()
```

#### 5. 事件驱动策略
- **核心逻辑：** 利用特定事件的价格反应
- **常见事件：** 财报发布/并购重组/指数调仓
- **适用市场：** 有明确事件催化的市场
- **风险：** 事件不及预期/市场反应异常

## 三、量化回测基础

### 什么是回测
用历史数据验证策略有效性

### 回测核心要素

| 要素 | 说明 | 注意事项 |
|------|------|----------|
| 数据 | 历史价格/财务/交易数据 | 需要复权/清洗 |
| 策略 | 明确的买卖规则 | 不能有模糊地带 |
| 初始资金 | 回测起始资金 | 影响仓位计算 |
| 交易成本 | 佣金/印花税/滑点 | 必须考虑，否则高估收益 |
| 回测周期 | 回测的时间范围 | 至少一个完整牛熊周期 |

### 回测核心陷阱

#### 1. 未来函数（Look-ahead Bias）
- **问题：** 使用了当时无法获得的信息
- **示例：** 用当日收盘价决定当日开盘买入
- **避免：** 确保决策只用历史数据

#### 2. 幸存者偏差（Survivorship Bias）
- **问题：** 只回测了存活到现在的股票
- **示例：** 回测沪深 300 成分股，忽略了被剔除的股票
- **避免：** 使用包含退市股票的历史数据

#### 3. 过度拟合（Overfitting）
- **问题：** 策略参数过度优化，只适配历史数据
- **示例：** 测试 1000 组参数，选最好的那组
- **避免：**
  - 样本外验证（Out-of-sample testing）
  - 参数敏感性分析
  - 逻辑简单优于复杂

#### 4. 忽略交易成本
- **问题：** 未考虑佣金/印花税/滑点
- **影响：** 高频策略收益被严重高估
- **避免：** 回测中加入 realistic 的交易成本

### 回测评估指标

| 指标 | 公式 | 含义 | 合理值 |
|------|------|------|--------|
| 总收益率 | (期末值 - 期初值) / 期初值 | 总收益 | - |
| 年化收益率 | (1+ 总收益率)^(1/年数) - 1 | 年化收益 | >15% 优秀 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动 | 风险调整后收益 | >1 合格，>2 优秀 |
| 最大回撤 | max(峰值 - 谷值) / 峰值 | 最大亏损幅度 | <20% 优秀 |
| 胜率 | 盈利次数 / 总次数 | 成功概率 | >50% 合格 |
| 盈亏比 | 平均盈利 / 平均亏损 | 赚赔比例 | >2 优秀 |
| 卡玛比率 | 年化收益 / 最大回撤 | 收益回撤比 | >2 优秀 |

## 四、量化策略开发流程

### 标准流程
```
1. 想法 → 2. 数据 → 3. 回测 → 4. 优化 → 5. 模拟 → 6. 实盘
```

### 详细步骤

#### 1. 策略想法
- 来源：学术研究/市场观察/经验总结
- 要求：逻辑清晰、可量化、可回测

#### 2. 数据准备
- 数据类型：价格/成交量/财务/宏观/另类数据
- 数据质量：完整性/准确性/一致性
- 数据处理：清洗/复权/对齐

#### 3. 策略回测
- 回测框架：Backtrader/PyAlgoTrade/聚宽/米筐
- 回测设置：初始资金/手续费/滑点
- 结果分析：收益/风险/稳定性

#### 4. 策略优化
- 参数优化：网格搜索/遗传算法
- 避免过拟合：样本外验证/交叉验证
- 稳健性检验：不同市场/不同周期

#### 5. 模拟交易
- 目的：验证实盘可行性
- 时间：至少 3-6 个月
- 对比：模拟 vs 回测，差异分析

#### 6. 实盘交易
- 小资金起步：验证实盘表现
- 逐步加仓：确认策略稳定
- 持续监控：策略是否失效

## 五、量化风险管理

### 风险类型

| 风险类型 | 说明 | 管理方法 |
|----------|------|----------|
| 市场风险 | 整体市场下跌 | 仓位控制/对冲 |
| 策略风险 | 策略失效 | 多策略分散 |
| 模型风险 | 模型错误/假设不成立 | 定期检验/更新 |
| 技术风险 | 系统故障/网络问题 | 冗余备份/监控 |
| 流动性风险 | 无法及时买卖 | 限制标的/仓位 |
| 操作风险 | 人为错误 | 自动化/复核 |

### 仓位管理方法

#### 1. 固定比例法
- 每笔交易固定比例（如 5%）
- 简单，但没考虑风险差异

#### 2. 凯利公式
```
f* = (p × b - q) / b
f* = 最优仓位比例
p = 胜率
q = 败率 (1-p)
b = 盈亏比
```
- 理论最优，但实际需打折使用

#### 3. 波动率调整法
- 波动大的标的仓位小
- 波动小的标的仓位大
- 使各标的贡献风险相等

#### 4. 最大回撤控制
- 设定最大回撤阈值（如 20%）
- 接近阈值时降低仓位
- 保护本金安全
