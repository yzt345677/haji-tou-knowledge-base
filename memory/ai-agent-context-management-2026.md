## 🧠 从"对话者"到"行动者"的演进

### 传统 Chatbot 模式（2023-2024）
- **交互形态：** 单轮问答或简单多轮对话
- **上下文：** 有限上下文窗口
- **工程核心：** 提示词优化
- **局限：** 模型接收单一输入，生成一次回应，交互即终止

### AI Agent 模式（2025-2026）
- **交互形态：** 复杂任务流，多轮协作
- **上下文：** 长期记忆 + 文件系统协作
- **工程核心：** 任务编排 + 记忆管理 + 工具调用
- **优势：** 自主规划、工具调用、记忆管理，实现从"对话"到"行动"的跃迁

## 📊 2026 年 AI Agent 核心能力

### 1. 规划能力（大脑）
**从零散指令到"任务流"：**

早期 LLM：
```
用户："帮我写个报告"
→ 生成一篇报告
```

AI Agent：
```
用户："帮我组织一场公司团建活动"
→ 拆解为：
  1. 确定日期 → 查询日历 → 协调参与人时间
  2. 选择场地 → 搜索场地 → 对比价格 → 预订
  3. 安排交通 → 查询路线 → 预订车辆
  4. 发送通知 → 生成邀请函 → 邮件/消息推送
  5. 跟踪反馈 → 收集回复 → 调整安排
```

---

### 2. 多轮对话上下文管理

**挑战：**
- 单 Agent 上下文窗口有限
- 长周期任务容易丢失前期信息
- 多主题切换时容易混淆

**解决方案：**

#### 方案 1：文件系统协作
- 将对话历史写入文件
- 关键决策点存档
- 下次对话时读取相关文件

#### 方案 2：长期记忆沉淀
- 使用向量数据库存储历史
- 语义检索相关上下文
- 动态加载到当前对话

#### 方案 3：多 Agent 协作
- 每个 Agent 负责特定子任务
- 通过文件系统传递上下文
- 避免单 Agent 上下文溢出

---

### 3. 记忆管理（核心突破）

**记忆类型：**
| 类型 | 用途 | 存储方式 |
|------|------|----------|
| 短期记忆 | 当前对话上下文 | 内存/会话窗口 |
| 工作记忆 | 任务执行状态 | 临时文件 |
| 长期记忆 | 经验/知识沉淀 | 向量数据库/文件 |
| 程序记忆 | 技能/工具调用 | 技能库 |

**记忆管理最佳实践：**
1. **分层存储** — 不同记忆类型用不同存储
2. **定期沉淀** — 短期→长期，raw→curated
3. **语义检索** — 向量检索 + 关键词匹配
4. **上下文压缩** — 摘要历史，保留核心

## 🎯 上下文管理技巧（给老大的建议）

### 技巧 1：会话命名
```bash
# 给会话命名，方便恢复
claude -n auth-refactor
# 或中途命名
/rename feature-payment-v2
# 之后精准恢复
claude --resume feature-payment-v2
```

### 技巧 2：硬性规则 vs 软性偏好
- **CLAUDE.md** — 硬性规则（代码风格、提交流程）
- **Auto Memory** — 软性偏好（回答格式、调试方法）

### 技巧 3：定期清理
- 归档过期会话
- 删除临时笔记
- 保持记忆系统精简

### 技巧 4：上下文压缩
- 长对话自动摘要
- 保留核心决策点
- 删除冗余过程
