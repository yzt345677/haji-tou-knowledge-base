## 📌 核心概述

**量子算法** 是利用量子力学特性（叠加态、纠缠、干涉）设计的计算方法，在特定问题上可实现相对于经典算法的**指数级或平方根级加速**。

**两大明星算法：**
- **Shor 算法**（1994）— 大数质因数分解，指数级加速
- **Grover 算法**（1996）— 无序数据库搜索，平方根加速

---

## 🔐 Shor 算法（彼得·肖尔，1994）

### 核心功能

**大整数质因数分解** — 找到一个大数的质因数。

**例如：**
- 经典计算机分解 15 = 3 × 5（简单）
- 但分解一个 2048 位的大数？经典计算机需要宇宙年龄那么久！

### 加速效果

| 指标 | 经典算法 | Shor 算法 |
|------|---------|---------|
| **时间复杂度** | 亚指数级增长 | 多项式时间 |
| **2048 位 RSA** | 约 10^39 次运算 | 约 200 万次运算 |
| **加速倍数** | 基准 | **亿亿亿倍** |

### 对密码学的影响

**RSA 加密原理：**
- 用两个大质数相乘生成公钥
- 安全性基于"分解大数极其困难"
- 银行/政府/互联网都在用

**Shor 算法的威胁：**
- 理论上可以破解 RSA 加密
- 一旦有足够大的量子计算机，现有加密体系将崩溃
- 这推动了**后量子密码学**的发展

### 算法原理（简化版）

**核心思想：** 将因数分解问题转化为**周期查找问题**。

**步骤：**
1. 随机选一个数 a，计算 a^x mod N 的周期 r
2. 用量子傅里叶变换（QFT）快速找到周期 r
3. 用经典数学从 r 推导出质因数

**关键技巧：**
- 量子并行性：同时计算所有可能的 x
- 量子干涉：放大正确答案，抵消错误答案
- 量子傅里叶变换：指数级加速周期查找

## 🔍 Grover 算法（洛夫·格罗弗，1996）

### 核心功能

**无序数据库搜索** — 在一堆数据中找到特定目标。

**例如：**
- 在 100 万人的电话簿中找一个人的号码
- 在密码空间中暴力破解密码
- 在组合优化问题中找最优解

### 加速效果

| 指标 | 经典算法 | Grover 算法 |
|------|---------|------------|
| **时间复杂度** | O(N) | O(√N) |
| **100 万数据** | 平均 50 万次操作 | 约 1000 次操作 |
| **加速倍数** | 基准 | **约 500 倍** |
| **1 亿数据** | 平均 5000 万次 | 约 1 万次 |

**平方根加速** 虽然不如 Shor 算法的指数级加速惊人，但**适用范围更广**喵～

### 应用场景

| 领域 | 应用 |
|------|------|
| **密码学** | 暴力破解密码（密钥空间搜索） |
| **数据库** | 无序数据快速检索 |
| **组合优化** | 旅行商问题/调度问题 |
| **图论** | 最短路径/最小生成树 |
| **数据挖掘** | 模式识别/异常检测 |
| **机器学习** | 特征选择/聚类优化 |

### 算法原理（简化版）

**核心思想：** 利用量子叠加和干涉，让目标项"自动凸显"。

**比喻：**
- 经典搜索 = 一个一个翻箱子找东西
- Grover 搜索 = 所有箱子同时打开，目标箱子发光

**步骤：**
1. **制备叠加态** — 所有可能解同时存在
2. **Oracle 操作** — 标记目标解（翻转相位）
3. **扩散操作** — 放大目标解的概率振幅
4. **重复迭代** — 约 √N 次后测量得到目标

**关键技巧：**
- 每次迭代让目标解的概率增加一点
- 约 √N 次迭代后，测量得到目标的概率接近 100%
- 再多迭代反而会降低概率（过犹不及！）

## 📊 两大算法对比

| 特性 | Shor 算法 | Grover 算法 |
|------|----------|------------|
| **发明年份** | 1994 | 1996 |
| **发明者** | 彼得·肖尔 | 洛夫·格罗弗 |
| **加速类型** | 指数级 | 平方根级 |
| **核心应用** | 大数分解 | 无序搜索 |
| **密码学影响** | 破解 RSA | 暴力破解密钥 |
| **量子资源** | 需要大量量子比特 | 相对较少 |
| **实用化难度** | 高（需要纠错） | 中（近期可能实现） |
| **适用范围** | 特定问题 | 广泛问题 |

## 🚀 2026 年应用进展

### Shor 算法现状

**技术挑战：**
- 需要大量逻辑量子比特（可能数千个）
- 需要量子纠错（2026 年中国突破"低于阈值"）
- 目前只能分解很小的数（实验阶段）

**2026 年进展：**
- 中科大潘建伟团队实现可扩展量子纠错
- "祖冲之三号"超导量子计算原型机问世
- 距离实用化还有距离，但方向明确

**应对策略：**
- **后量子密码学（PQC）** 正在部署
- 美国 NIST 已标准化多种抗量子算法
- 中国也在推进自主可控的后量子密码标准

### Grover 算法现状

**技术优势：**
- 需要的量子比特较少
- 对纠错要求相对较低
- 可能在近期量子计算机上实现

**2026 年应用探索：**
- 组合优化问题（物流/调度）
- 机器学习加速（特征选择/聚类）
- 金融投资组合优化
- 药物分子筛选

**实际限制：**
- 需要量子 RAM（qRAM）才能高效加载经典数据
- 目前数据加载是瓶颈
- 混合量子 - 经典算法是过渡方案

## 💡 其他重要量子算法

### 量子相位估计（QPE）

**功能：** 估算量子态的本征值
**应用：** Shor 算法的核心子程序、量子化学模拟

### 变分量子本征求解器（VQE）

**功能：** 求解矩阵本征值（如分子能量）
**应用：** 量子化学、材料科学、药物研发
**优势：** 适合含噪声中等规模量子（NISQ）设备

### 量子近似优化算法（QAOA）

**功能：** 解决组合优化问题
**应用：** 旅行商问题、图分割、最大割问题

### TensorFlow Quantum（TFQ）

**功能：** 混合量子 - 经典机器学习框架
**应用：** 量子神经网络、量子强化学习
**开发者：** Google

## 🎓 量子算法的数学基础

**Grover 算法查询次数：** O(√N)，其中 N 为搜索空间大小

**示例计算：**
- 1024 个数据 → 经典需要 512 次，Grover 需要 32 次
- 100 万个数据 → 经典需要 50 万次，Grover 需要 1000 次
- 1 亿个数据 → 经典需要 5000 万次，Grover 需要 1 万次

**Shor 算法复杂度：** O((log N)³)，其中 N 为待分解的数
