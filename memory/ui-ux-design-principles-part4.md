## 📐 常见设计模式（Landing Page Patterns）

### 一、24 种落地页模式详解

#### 1. Hero-Centric（英雄区主导）

**结构：**
```
┌─────────────────────────────┐
│   Hero Section (100vh)      │
│   - 大标题                  │
│   - 副标题                  │
│   - 主 CTA 按钮             │
│   - 英雄图/视频             │
└─────────────────────────────┘
```

**适用场景：**
- SaaS 产品发布
- 应用下载页
- 服务介绍页

**关键元素：**
- 清晰的价值主张（一句话说明白）
- 单个主 CTA（不要多个选择）
- 高质量视觉（图/视频）
- 社会证明（Logo 墙/用户数）

**示例文案：**
```
标题：让团队协作效率提升 300%
副标题：一站式项目管理工具，让工作更简单
CTA：免费开始使用 →
社会证明：已有 10,000+ 团队选择我们
```

#### 2. Problem-Solution（问题 - 解决方案）

**结构：**
```
┌─────────────────────────────┐
│   Hero (简要价值主张)        │
├─────────────────────────────┤
│   Problem Section           │
│   - 痛点 1 + 插图           │
│   - 痛点 2 + 插图           │
│   - 痛点 3 + 插图           │
├─────────────────────────────┤
│   Solution Section          │
│   - 功能 1 → 解决痛点 1      │
│   - 功能 2 → 解决痛点 2      │
│   - 功能 3 → 解决痛点 3      │
├─────────────────────────────┤
│   CTA                       │
└─────────────────────────────┘
```

**适用场景：**
- B2B 服务
- 咨询服务
- 健康/健身产品

**写作技巧：**
- 痛点要具体（"每天加班 3 小时" 而不是 "效率低"）
- 解决方案要直接对应痛点
- 用数据支撑（"节省 50% 时间"）

---

#### 3. Feature-Benefit（功能 - 收益）

**结构：**
```
┌─────────────────────────────┐
│   Hero                      │
├─────────────────────────────┤
│   Feature 1                 │
│   [图] ←→ 文字说明          │
│   - 功能描述                │
│   - 用户收益                │
├─────────────────────────────┤
│   Feature 2                 │
│   文字说明 → [图]           │
│   - 功能描述                │
│   - 用户收益                │
├─────────────────────────────┤
│   ... (3-5 个功能)           │
├─────────────────────────────┤
│   CTA                       │
└─────────────────────────────┘
```

**适用场景：**
- 复杂产品
- 技术产品
- 多功能平台

**写作技巧：**
- 功能是"是什么"，收益是"得到什么"
- 功能：「实时同步」
- 收益：「随时随地访问最新数据」

---

#### 4. Social Proof（社会证明）

**结构：**
```
┌─────────────────────────────┐
│   Hero + 用户数/评分        │
├─────────────────────────────┤
│   Logo 墙 (知名客户)         │
├─────────────────────────────┤
│   用户评价 (3-5 条)          │
│   - 头像 + 姓名 + 职位      │
│   - 评价内容 + 评分         │
├─────────────────────────────┤
│   案例研究 (2-3 个)          │
├─────────────────────────────┤
│   媒体提及                  │
├─────────────────────────────┤
│   CTA                       │
└─────────────────────────────┘
```

**适用场景：**
- 成熟产品
- B2B 服务
- 高客单价产品

**关键元素：**
- 真实用户头像（不要用素材图）
- 具体数据（"效率提升 47%"）
- 可验证的评价（带公司/职位）

---

#### 5. Pricing-Focused（定价主导）

**结构：**
```
┌─────────────────────────────┐
│   Hero (价值主张)            │
├─────────────────────────────┤
│   Pricing Tables (3 档)      │
│   ┌───┬───┬───┐            │
│   │基 │专 │企 │            │
│   │础 │业 │业 │            │
│   └───┴───┴───┘            │
├─────────────────────────────┤
│   FAQ (定价相关)             │
├─────────────────────────────┤
│   CTA                       │
└─────────────────────────────┘
```

**适用场景：**
- SaaS 产品
- 订阅服务
- 会员制度

**定价心理学：**
- 3 档定价（中间档最畅销）
- 锚定效应（高价档衬托中间档）
- 推荐标识（"最受欢迎"）
- 年付优惠（显示省多少钱）

---

#### 6. Storytelling（故事叙述）

**结构：**
```
┌─────────────────────────────┐
│   Hero (引入故事)            │
├─────────────────────────────┤
│   Chapter 1: 起源故事       │
├─────────────────────────────┤
│   Chapter 2: 挑战           │
├─────────────────────────────┤
│   Chapter 3: 突破           │
├─────────────────────────────┤
│   Chapter 4: 使命           │
├─────────────────────────────┤
│   CTA                       │
└─────────────────────────────┘
```

**适用场景：**
- 创始人品牌
- 社会企业
- 众筹项目

**写作技巧：**
- 真实情感（不要过度包装）
- 冲突与解决（故事需要张力）
- 读者代入（"你也会遇到..."）

---

#### 7. Comparison（对比模式）

**结构：**
```
┌─────────────────────────────┐
│   Hero (我们是更好的选择)    │
├─────────────────────────────┤
│   Comparison Table          │
│   ┌─────────┬─────┬─────┐  │
│   │ 功能    │ 我们│ 他们│  │
│   ├─────────┼─────┼─────┤  │
│   │ 功能 1  │ ✅  │ ❌  │  │
│   │ 功能 2  │ ✅  │ ⚠️  │  │
│   └─────────┴─────┴─────┘  │
├─────────────────────────────┤
│   CTA                       │
└─────────────────────────────┘
```

**适用场景：**
- 挑战市场领导者
- 差异化明显的产品
- 新品类教育

**注意事项：**
- 对比要真实（不要虚假宣传）
- 聚焦自身优势（不要过度贬低）
- 用数据说话（不要主观评价）

---

#### 8. Demo-First（演示优先）

**结构：**
```
┌─────────────────────────────┐
│   Hero + 产品演示视频/GIF   │
│   (自动播放/无声)           │
├─────────────────────────────┤
│   功能亮点 (配合演示)        │
├─────────────────────────────┤
│   用户评价                  │
├─────────────────────────────┤
│   CTA (免费试用)            │
└─────────────────────────────┘
```

**适用场景：**
- 视觉化产品
- 交互型产品
- 设计工具

**关键元素：**
- 演示要短（15-30 秒）
- 展示核心价值（不要功能罗列）
- 无声播放（考虑办公环境）

### 二、核心区块设计（Section Patterns）

#### Hero Section（英雄区）

**必备元素：**
```
1. 主标题（H1）- 一句话价值主张
2. 副标题 - 补充说明
3. 主 CTA - 单个明确行动
4. 视觉元素 - 产品图/插图/视频
5. 社会证明 - 可选（用户数/Logo/评分）
```

**最佳实践：**
- 标题不超过 10 个字
- 副标题不超过 25 个字
- CTA 文案要具体（"免费开始" 而不是 "提交"）
- 视觉与文案一致

---

#### Features Section（功能区）

**布局选项：**
```
A. 网格布局 (2x2 或 3x2)
B. 交替布局 (左图右文 / 右图左文)
C. 卡片布局 (3-4 列)
D. 列表布局 (带图标)
```

**写作公式：**
```
功能名称 + 简短描述 + 用户收益

例：
「实时同步」
数据自动保存并同步到所有设备
→ 随时随地访问最新信息
```

---

#### Testimonials Section（评价区）

**高质量评价要素：**
```
✅ 真实头像（不要用素材图）
✅ 完整姓名 + 职位/公司
✅ 具体成果（"效率提升 47%"）
✅ 使用场景（"我们团队用它来..."）
❌ 避免：模糊评价（"很好用"）
```

**布局建议：**
- 3 条评价最佳（奇数原则）
- 轮播 vs 平铺（内容多时轮播）
- 视频评价 > 文字评价

---

#### Pricing Section（定价区）

**定价表设计：**
```
┌───────────────────┐
│   方案名称        │
│   $XX/月         │
│   ────────────   │
│   ✅ 功能 1       │
│   ✅ 功能 2       │
│   ✅ 功能 3       │
│   ────────────   │
│   [CTA 按钮]      │
└───────────────────┘
```

**最佳实践：**
- 3 档定价（基础/专业/企业）
- 中间档标记"最受欢迎"
- 年付显示优惠（"省 20%"）
- 企业档写"联系我们"

---

#### FAQ Section（常见问题）

**问题选择：**
- 购买障碍（"可以退款吗？"）
- 技术疑问（"支持 XX 吗？"）
- 价格疑虑（"为什么比 XX 贵？"）
- 使用问题（"需要安装吗？"）

**设计形式：**
```
▼ 问题标题（可展开）
  详细回答（展开后显示）
```

---

#### CTA Section（行动号召）

**高效 CTA 要素：**
```
1. 明确行动（"免费开始"）
2. 降低风险（"无需信用卡"）
3. 紧迫感（"限时优惠"）- 慎用
4. 社会证明（"10,000+ 已加入"）
```

**位置建议：**
- Hero 区（主 CTA）
- 功能介绍后
- 评价区后
- 页面底部（最终 CTA）

## ✅ 完整检查清单（Pre-delivery Checklist）

### 一、基础检查（Basic）

#### 视觉一致性

- [ ] **颜色使用一致** — 全站不超过 5 种主色
- [ ] **字体统一** — 不超过 3 种字体家族
- [ ] **间距规律** — 使用 8px 网格系统
- [ ] **圆角一致** — 所有圆角统一（4px/8px/12px）
- [ ] **阴影统一** — 阴影风格一致（软/硬）

#### 内容检查

- [ ] **无错别字** — 拼写检查
- [ ] **语法正确** — 语句通顺
- [ ] **链接有效** — 所有链接可点击
- [ ] **图片加载** — 所有图片正常显示
- [ ] **文案一致** — 术语使用统一

#### 功能检查

- [ ] **表单可用** — 提交成功/失败有反馈
- [ ] **按钮响应** — 悬停/点击状态
- [ ] **导航正常** — 所有菜单项可访问
- [ ] **搜索功能** — 搜索结果准确
- [ ] **动画流畅** — 无卡顿/闪烁

---

### 二、无障碍检查（Accessibility）

#### WCAG 2.1 AA 合规

- [ ] **文本对比度 ≥ 4.5:1** — 普通文本
- [ ] **大文本对比度 ≥ 3:1** — 18px 以上
- [ ] **焦点状态可见** — 键盘导航时
- [ ] **所有功能可键盘访问** — 无需鼠标
- [ ] **图片有 alt 文本** — 描述性文字
- [ ] **表单有 label** — 关联标签
- [ ] **错误信息清晰** — 说明问题 + 解决方案
- [ ] **支持放大 200%** — 不失真/不溢出
- [ ] **减少动画选项** — 尊重 prefers-reduced-motion
- [ ] **颜色不是唯一信息** — 有文字/图标辅助

#### 快速测试方法

```bash
# 1. 键盘测试
- 按 Tab 键遍历所有交互元素
- 检查焦点是否可见
- 检查是否能完成所有操作

# 2. 屏幕阅读器测试
- 用 VoiceOver (Mac) 或 NVDA (Windows)
- 检查内容顺序是否合理
- 检查图片是否有描述

# 3. 对比度测试
- 用 Chrome DevTools 的 Lighthouse
- 或用 WebAIM Contrast Checker
```

### 三、性能检查（Performance）

#### Core Web Vitals

| 指标 | 目标 | 测量方法 |
|------|------|---------|
| **LCP** (最大内容绘制) | < 2.5s | PageSpeed Insights |
| **FID** (首次输入延迟) | < 100ms | PageSpeed Insights |
| **CLS** (累积布局偏移) | < 0.1 | PageSpeed Insights |

#### 图片优化

- [ ] **使用 WebP 格式** — 比 JPG 小 30%
- [ ] **适当压缩** — 质量 80-85%
- [ ] **懒加载** — 视口外图片延迟加载
- [ ] **指定尺寸** — 避免布局偏移
- [ ] **响应式图片** — srcset 适配多设备

#### 代码优化

- [ ] **CSS 压缩** — 移除空白和注释
- [ ] **JS 压缩** — 使用 Terser 等工具
- [ ] **代码分割** — 按需加载
- [ ] **移除未使用代码** — Tree Shaking
- [ ] **缓存策略** — 静态资源缓存

#### 加载优化

- [ ] **关键 CSS 内联** — 首屏样式直接嵌入
- [ ] **非关键 JS 延迟** — defer/async
- [ ] **预加载关键资源** — `<link rel="preload">`
- [ ] **使用 CDN** — 静态资源分发
- [ ] **启用 Gzip/Brotli** — 压缩传输

### 四、响应式检查（Responsive）

#### 断点测试

| 断点 | 宽度 | 测试内容 |
|------|------|---------|
| **手机小** | 375px | iPhone SE |
| **手机大** | 414px | iPhone 14 Pro Max |
| **平板** | 768px | iPad |
| **桌面小** | 1024px | 小屏笔记本 |
| **桌面大** | 1440px | 标准桌面 |
| **桌面超大** | 1920px | 大屏显示器 |

#### 响应式检查项

- [ ] **文字不换行** — 小屏幕文字可读
- [ ] **按钮够大** — 最小 44x44px 点击区域
- [ ] **图片适配** — 不溢出/不变形
- [ ] **导航适配** — 汉堡菜单/底部导航
- [ ] **表单适配** — 输入框宽度合适
- [ ] **表格适配** — 横向滚动/卡片化
- [ ] **间距适配** — 小屏幕适当缩小

---

### 五、SEO 检查（Search Engine Optimization）

#### 基础 SEO

- [ ] **标题标签** — 每页唯一，包含关键词
- [ ] **Meta 描述** — 150-160 字符
- [ ] **H1 标签** — 每页一个
- [ ] **标题层级** — H1 → H2 → H3 有序
- [ ] **图片 alt** — 描述性文字
- [ ] **内部链接** — 相关页面互链
- [ ] **URL 结构** — 清晰有意义

#### 技术 SEO

- [ ] **sitemap.xml** — 提交搜索引擎
- [ ] **robots.txt** — 正确配置
- [ ] **结构化数据** — Schema.org 标记
- [ ] **Canonical 标签** — 避免重复内容
- [ ] **404 页面** — 自定义错误页
- [ ] **HTTPS** — 全站加密

## 🚫 行业避坑指南（详细版）

### 金融行业避坑

**❌ 避免：**
- AI 紫色/粉色渐变（不专业）
- 过于花哨的动画（影响信任感）
- 不清晰的数字展示（金融数据需精确）
- 缺少安全标识（SSL/合规认证）
- 复杂的导航（用户需要快速找到信息）

**✅ 推荐：**
- 稳重的蓝/金色系
- 清晰的数字和表格
- 安全认证标识显眼
- 简洁直接的导航
- 客服入口明显

---

### 医疗健康避坑

**❌ 避免：**
- 暗黑色系（压抑感）
- 过于鲜艳的颜色（不专业）
- 复杂的导航（用户可能紧急）
- 缺少紧急联系信息
- 医学术语过多（用户看不懂）

**✅ 推荐：**
- 清洁的蓝/白色系
- 简洁直接的导航
- 紧急联系信息突出
- 通俗易懂的文案
- 医生资质展示

---

### 奢侈品避坑

**❌ 避免：**
- 过于拥挤的布局（缺少呼吸感）
- 廉价的颜色组合
- 过多的促销元素
- 低质量图片
- 明显的折扣标识

**✅ 推荐：**
- 大量留白
- 黑/白/金经典配色
- 高质量产品图片
- 精致的细节设计
- 稀缺感营造

---

### 儿童/教育避坑

**❌ 避免：**
- 过于严肃的设计
- 小字体和复杂交互
- 缺乏视觉反馈
- 缺少家长控制
- 过多的文字说明

**✅ 推荐：**
- 鲜艳活泼的颜色
- 大按钮和简单交互
- 即时反馈和鼓励
- 家长入口/控制
- 图示多于文字

---

### SaaS/科技避坑

**❌ 避免：**
- 功能罗列（不说价值）
- 缺少定价信息
- 没有免费试用
- 技术术语过多
- 缺少社会证明

**✅ 推荐：**
- 价值主张清晰
- 定价透明
- 免费试用/演示
- 通俗易懂的文案
- 用户评价/案例
