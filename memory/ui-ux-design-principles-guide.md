## 🌈 161 种配色方案（按行业）

### 配色选择原则

1. **行业匹配** — 金融用蓝/金，医疗用蓝/绿，环保用绿/棕
2. **品牌一致性** — 主色 + 辅色 + 强调色不超过 3 种
3. **可访问性** — 文本对比度至少 4.5:1（WCAG AA）
4. **情感共鸣** — 颜色传递正确情感（信任/活力/平静）

## 🔤 57 种字体组合

### 字体选择原则

1. **可读性优先** — 正文用易读字体，标题可用装饰字体
2. **风格匹配** — 字体传递正确情感（专业/友好/创意）

## ✅ 交付前检查清单（Pre-delivery Checklist）

### 基础检查

- [ ] **无 Emoji 作为图标** — 使用 SVG 图标（Heroicons/Lucide）
- [ ] **cursor-pointer** — 所有可点击元素都有指针光标
- [ ] **悬停状态** — 平滑过渡（150-300ms）
- [ ] **对比度** — 浅色模式文本对比度至少 4.5:1
- [ ] **焦点状态** — 键盘导航时焦点可见
- [ ] **减少动画偏好** — 尊重 prefers-reduced-motion
- [ ] **响应式** — 测试 375px、768px、1024px、1440px

### 无障碍检查（Accessibility）

- [ ] **ARIA 标签** — 所有交互元素有适当 ARIA 标签
- [ ] **键盘导航** — 所有功能可键盘访问
- [ ] **屏幕阅读器** — 内容对屏幕阅读器友好
- [ ] **颜色对比** — 符合 WCAG AA 标准
- [ ] **焦点顺序** — 逻辑的焦点顺序
- [ ] **替代文本** — 所有图片有 alt 文本

### 性能检查

- [ ] **图片优化** — WebP 格式、适当压缩、懒加载
- [ ] **代码分割** — 按需加载、减少初始包大小
- [ ] **缓存策略** — 静态资源适当缓存
- [ ] **首屏加载** — 关键内容优先加载
- [ ] **Core Web Vitals** — LCP < 2.5s, FID < 100ms, CLS < 0.1

## 🚫 行业避坑指南（Anti-patterns）

### 金融行业

**避免：**
- ❌ AI 紫色/粉色渐变（不专业）
- ❌ 过于花哨的动画（影响信任感）
- ❌ 不清晰的数字展示（金融数据需精确）

**推荐：**
- ✅ 稳重的蓝/金色系
- ✅ 清晰的数字和表格
- ✅ 安全认证标识

### 医疗健康

**避免：**
- ❌ 暗黑色系（压抑感）
- ❌ 过于鲜艳的颜色（不专业）
- ❌ 复杂的导航（用户可能紧急）

**推荐：**
- ✅ 清洁的蓝/白色系
- ✅ 简洁直接的导航
- ✅ 紧急联系信息突出

### 奢侈品

**避免：**
- ❌ 过于拥挤的布局（缺少呼吸感）
- ❌ 廉价的颜色组合
- ❌ 过多的促销元素

**推荐：**
- ✅ 大量留白
- ✅ 黑/白/金经典配色
- ✅ 高质量产品图片

### 儿童/教育

**避免：**
- ❌ 过于严肃的设计
- ❌ 小字体和复杂交互
- ❌ 缺乏视觉反馈

**推荐：**
- ✅ 鲜艳活泼的颜色
- ✅ 大按钮和简单交互
- ✅ 即时反馈和鼓励

## 💡 哈基偷的学习心得
